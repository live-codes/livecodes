# Headless Mode

import LiveCodes from '../../src/components/LiveCodes.tsx';

The LiveCodes [SDK](../sdk/index.mdx) can be used to create playgrounds in headless mode. In this mode, no visible output is displayed in the embedding web page. However, all [SDK methods](../sdk/js-ts.mdx#sdk-methods) are accessible (e.g. for [updating code](./js-ts.mdx#setconfig), [getting compiled code](./js-ts.mdx#getcode), console output, [result HTML](./js-ts.mdx#getcode), [shareable URLs](./js-ts.mdx#getshareurl), [formatting code](./js-ts.mdx#format), [running tests](./js-ts.mdx#runtests), etc).

This provides the power of leveraging the wide range of features and language support offered by LiveCodes, while retaining full control over the UI.

## Usage

To create a headless playground, set the [embed option](./js-ts.mdx#embed-options) [`headless`](../sdk/js-ts.mdx#headless) to `true`.

Please note that in headless mode, the first parameter (`container`) of the function [`createPlayground`](../sdk/js-ts.mdx#createplayground) is optional and can be omitted.

<div style={{ clear: 'both' }}></div>

Example:

```js
import { createPlayground } from 'livecodes';

createPlayground({
  view: 'headless',
  config: {
    markup: {
      language: 'markdown',
      content: '# Hello World!',
    },
  },
}).then(async (playground) => {
  const code = await playground.getCode();
  console.log(code.markup.compiled); // "<h1>Hello World!</h1>"
  console.log(code.result); // (result page HTML)
});
```

## Examples

The following examples show how to use the headless mode to make a Markdown editor, an MDX editor and a Python interpreter.

:::tip

You may want to view the following playgrounds in full screen (using the full screen button in the top right of each playground).

:::

### Markdown Editor

In this demo, code changes are watched using the SDK method [`watch('code', callback)`](./js-ts.mdx#watch). The callback function accepts an argument which is an object with the properties `code` and `config` (see [`getCode`](./js-ts.mdx#getcode) and [`getConfig`](./js-ts.mdx#getconfig)). The compiled code is obtained as `code.markup.compiled`.

### MDX Editor

In this demo, code changes are watched using the SDK method [`watch('code', callback)`](./js-ts.mdx#watch). The callback function accepts an argument which is an object with the properties `code` and `config` (see [`getCode`](./js-ts.mdx#getcode) and [`getConfig`](./js-ts.mdx#getconfig)). The result HTML is obtained as `code.result`.

:::tip

If you do not want to run the result page in the headless playground and only want to get the generated result HTML, you can set the configuration option [`autoupdate](../configuration/configuration-object.mdx#autoupdate) to `false`.

:::

### Python Interpreter

In this demo, console output is obtained using the SDK method [`watch('code', callback)`](./js-ts.mdx#watch). The callback function accepts an argument which is an object with the properties `method` and `args` indicating the console method and the arguments that were passed (as an array).
